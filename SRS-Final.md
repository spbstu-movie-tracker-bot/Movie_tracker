# Требования к спецификации Movie Tracker and Advisor Bot

## 1. Введение
Данный документ описывает требования к разработке Telegram-бота **«Movie Tracker and Advisor Bot»**.

### 1.1. Цель проекта
Этот проект представляет собой бота в Telegram, для поиска фильмов, добавления их в список просмотра и получения рекомендаций по фильмам.
Для лучшего понимания задачи и сбора идей по реализации функционала были проанализированы примеры аналогичных ботов, что позволило определить оптимальный набор возможностей.

### 1.2. Основные функции бота
Бот предоставляет следующие ключевые возможности:
* **Поиск фильмов по названию.** Для обеспечения эффективного поиска система ищет фильмы, названия которых содержат слова, введённые пользователем, и возвращает список найденных фильмов, отсортированных по популярности. После выбора конкретного фильма отображается полная информация о нём: название, постер, дата выхода, рейтинг и описание. Пользователь может вводить название фильма непосредственно в чат без использования специальных команд, что делает процесс поиска более удобным.
* **Добавление фильма в список watchlist.** После поиска фильма результаты отображаются в виде списка с интерактивными кнопками. Пользователь выбирает фильм нажатием на кнопку, что автоматически добавляет его в личный список отслеживания — без необходимости вручную вводить идентификатор фильма.
* **Просмотр списка отслеживания** с помощью команды `/watchlist`.
* **Удаление фильма из списка отслеживания.** Чтобы удалить фильм из списка просмотра, следует нажать кнопку «Удалить» рядом с каждым фильмом в списке.
* **Получение рекомендаций фильмов** с помощью команды `/recommend`, которая принимает два необязательных параметра: жанр и год выпуска. Если параметры указаны, бот фильтрует фильмы по этим критериям и возвращает результаты, отсортированные по популярности. Если параметры не заданы, бот возвращает наиболее популярные фильмы в целом.
* **Отображение справки и списка доступных команд** с помощью команды `/help`.
* **Приветственный экран и краткое введение** при использовании команды `/start`.

## 2. Общие требования к проекту

### 2.1. Репозиторий и система контроля версий
Исходный код проекта должен храниться в системе контроля версий Git. Репозиторий проекта размещён по адресу:
`https://github.com/spbstu-movie-tracker-bot`

Структура репозитория должна включать:
* Корневую директорию с основным кодом бота
* Файл `README.md` с описанием проекта и инструкциями по запуску
* Файл `requirements.txt` или `pyproject.toml` с зависимостями проекта
* Конфигурационные файлы для Docker
* Файл `.gitignore` для исключения служебных файлов

### 2.2. Система сборки и запуска
Проект должен поддерживать следующие способы запуска:
* **Локальный запуск:** Возможность запуска бота на локальной машине разработчика.
* **Запуск через Docker:** Основной способ развёртывания проекта с использованием контейнеризации.
Должна быть предоставлена документация с пошаговыми инструкциями по запуску проекта обоими способами.

### 2.3. Контейнеризация
Проект должен быть полностью контейнеризован с использованием Docker. Требования к контейнеризации:
* **Dockerfile:** Должен содержать инструкции по сборке образа Docker для приложения.
* **Docker Compose:** Необходимо использовать файл `docker-compose.yml`, поскольку проект включает в себя несколько сервисов (бот и база данных).
* **Образ Docker:** Готовый образ должен быть скомпилирован и загружен на Docker Hub.
* **Переменные среды:** Конфиденциальные данные (токены API, ключи доступа) должны передаваться через переменные среды, а не храниться в коде.

## 3. Функциональные требования

### 3.1. Аутентификация пользователей
**Процесс регистрации:**
Бот должен быть доступен для использования любым пользователем Telegram без необходимости предварительной регистрации.
* Пользователь находит бота в Telegram и нажимает кнопку `/start`.
* При первом запуске бот автоматически создаёт профиль пользователя в системе.
* Бот отображает приветственное сообщение с кратким описанием функционала и доступных команд.

**Идентификация пользователя:**
Идентификация пользователей осуществляется на основе уникального идентификатора пользователя в мессенджере, который автоматически предоставляется при взаимодействии пользователя с ботом.

**Хранение данных пользователя:**
Для каждого пользователя создаётся запись, содержащая его уникальный идентификатор и связанные с ним данные (списки фильмов, оценки, настройки).

### 3.2. Поиск и просмотр фильмов
**Механизм поиска:**
Бот должен предоставлять пользователям возможность поиска фильмов по названию. Для обеспечения эффективного поиска система осуществляет поиск фильмов, названия которых содержат слова, введённые пользователем. Пользователь может вводить название фильма непосредственно в чат без необходимости использования специальных команд.
При выполнении поискового запроса система осуществляет поиск по внешней базе данных фильмов. Результаты поиска возвращаются в виде списка фильмов, отсортированных по показателю популярности. Популярность представляет собой комплексную метрику, рассчитываемую на основе количества просмотров, добавлений в списки отслеживания, голосов пользователей и даты выхода фильма.

**Отображение результатов поиска:**
После ввода поискового запроса пользователь получает список возможных результатов, каждый из которых содержит искомые слова. Список отсортирован по убыванию популярности.
Результаты поиска отображаются в виде интерактивного списка с использованием встроенных кнопок. Каждая кнопка соответствует одному найденному фильму и отображает его название. При нажатии на кнопку пользователь получает сообщение с информацией о фильме и его подробностями.

**Ограничения и обработка ошибок при поиске:**
Если по текстовому запросу пользователя ничего не найдено, система должна отправить уведомление: «По вашему запросу фильмы не найдены».
Если пользователь отправляет медиафайлы (картинки, стикеры, голосовые сообщения или документы), система не обрабатывает их и возвращает сообщение: «Пожалуйста, отправьте текстовый запрос». При недоступности внешней базы данных фильмов бот уведомляет пользователя о временных технических неполадках.

**Детальная информация о фильме:**
После выбора конкретного фильма из списка результатов поиска бот отображает полную информацию о нём в формате единого сообщения. Детальная информация включает:
* Название фильма (оригинальное и локализованное)
* Постер (изображение обложки фильма)
* Дата выхода (год и точная дата премьеры)
* Рейтинг (средняя оценка пользователей)
* Описание (краткий синопсис сюжета)
* Жанры и продолжительность

Формат вывода обеспечивает удобное восприятие информации: ответное сообщение состоит из постера фильма, под которым располагается название жирным шрифтом, год выпуска и описание. Под сообщением располагается Inline-кнопка «Добавить в список».

### 3.3. Watchlist
**Добавление фильмов в список:**
В конце сообщения с информацией о фильме система отображает интерактивную кнопку «Добавить в список». При нажатии на эту кнопку фильм автоматически сохраняется в персональный список отслеживания пользователя.

**Просмотр списка отслеживания:**
Для просмотра своего списка фильмов пользователь использует команду `/watchlist`. Бот отображает полный список сохранённых фильмов. Каждый элемент списка содержит основную информацию о фильме (название, год выпуска).

**Удаление фильмов из списка:**
При просмотре списка рядом с каждым фильмом появляется кнопка удаления. Это избавляет пользователя от необходимости вручную вводить идентификатор фильма.

### 3.4. Персональные рекомендации
**Команда получения рекомендаций:**
Бот предоставляет возможность получения рекомендаций с помощью команды `/recommend`. Команда принимает два необязательных параметра: жанр и год выпуска.
* Параметр «жанр» позволяет указать интересующий жанр (боевик, комедия и т.д.).
* Параметр «год» позволяет ограничить результаты фильмами определённого года.

**Алгоритм формирования рекомендаций:**
При наличии параметров фильтрации бот запрашивает во внешней базе данных фильмы, соответствующие заданным критериям. Полученные результаты сортируются по показателю популярности в порядке убывания. Если пользователь не указывает параметры, бот возвращает список наиболее популярных фильмов в целом.

**Формат вывода рекомендаций:**
Результаты отображаются в виде списка фильмов с интерактивными кнопками, аналогично результатам поиска.

### 3.5. Сценарии использования (Use Cases)
Для лучшего понимания работы системы ниже представлены основные пользовательские сценарии:
* **Сценарий 1 (Поиск и добавление):** Я как пользователь хочу отправить боту название фильма, увидеть список результатов, выбрать нужный вариант и нажать кнопку «Добавить в список», чтобы не забыть посмотреть его на выходных.
* **Сценарий 2 (Рекомендации):** Я как пользователь хочу ввести команду с параметром жанра (например, комедия), чтобы бот прислал мне популярные фильмы в этом жанре на вечер.
* **Сценарий 3 (Некорректный ввод):** Я как пользователь случайно отправил боту стикер вместо названия фильма, и хочу получить понятное сообщение о том, что бот принимает только текстовые запросы.

## 4. Внешние зависимости и технологический стек

### 4.1. Источник данных о фильмах
**Выбор API:** В качестве источника данных о фильмах используется API сервиса TMDb (The Movie Database). TMDb API предоставляет все необходимые данные для реализации функционала бота и обладает качественной документацией.
**Характеристики TMDb API:** TMDb API представляет собой RESTful веб-сервис. Для работы требуется получение API-ключа.
**Ограничения API:** Стандартные ограничения для бесплатного использования API составляют 40 запросов в 10 секунд.
**Получаемые данные:** Название, описание сюжета, дата выхода, продолжительность, жанры, рейтинг, показатель популярности, постеры. API поддерживает функции поиска, фильтрации и детальной информации.

### 4.2. Язык программирования и фреймворки
Проект разрабатывается на языке Java с использованием фреймворка Spring Boot. Для обработки асинхронных операций используется Spring WebFlux.

### 4.3. База данных
В качестве основной базы данных используется PostgreSQL.

### 4.4. Документирование API
Для документирования REST API используется Spring Rest Docs.

### 4.5. Очередь сообщений
Для асинхронной обработки задач и обеспечения надёжной доставки сообщений используется RabbitMQ.

### 4.6. Инструменты тестирования
Проект включает модульные, интеграционные и сквозные тесты. Используются: JUnit для модульного тестирования, Mockito для создания mock-объектов, и TestContainers для интеграционного тестирования с реальными сервисами (PostgreSQL, RabbitMQ).

## 5. Список команд бота
* `/start` — инициализация работы с ботом, создание профиля, приветствие.
* `/help` — отображение справочной информации и списка команд.
* `/watchlist` — просмотр личного списка фильмов, которые пользователь хочет посмотреть.
* `/recommend [genre] [year]` — получение рекомендаций (например, `/recommend action 2020`).
* **Поиск фильмов** — ввод названия непосредственно в чат (без специальной команды).
